2019.08.01

useEffect hook 이름처럼 side effect를 실행하는 코드를 작성하는 hook이다. useEffect의 특징 가운데 한가지는 컴포넌트의 생애주기 별로 나눠 생각하지 않고, 렌더링 된 후 실행된다는 점이다. 컴포넌트가 언마운트될 때 실행되는 함수를 작성하려면 useEffect의 리턴값으로 함수를 넘긴다. 다른 훅과 마찬가지로 useEffect는 함수 컴포넌트 안에서만 사용되어야한다. 클래스 컴포넌트에서는 사용될 수 없다. 그리고 일반 함수에서는 사용될 수 없다.

2019.07.31

react context api context api 를 활용하면 여러 컴포넌트들 끼리 명시적으로 props를 통해 데이터를 넘겨주지 않고도 값을 공유할 수 있다. context 생성하면 Provider 와 Consumer 크게 두 가지 역할로 나뉜다. Provider는 값을 context를 구독하는 컴포넌트들에게 context의 변화를 알리는 역할 Consumer는 context의 변화를 구독하는 컴포넌트. 함수형 컴포넌트에서 컨텍스트에 접근하는 데 사용한다. 클래스 컴포넌트에서는 consumer를 대신하는 녀석이 <클래스 컴포넌트명>.contextType이다. 이렇게 지정하면 this.context 로 컨텍스트에 접근할 수 있다.

리엑트 클래스 컴포넌트에서 한 요소에 직접 접근할때 ref를 사용한다. 부모 컴포넌트에서 자식 컴포넌트에 접근할때는 props를 이용하지만, 직접 접근하고 싶은 특별한 경우에 사용. 나는 carousel slider 써드파티 라이브러리를 사용하다가 직접 왼쪽, 오른쪽 슬라이드 넘기는 버튼을 만들어야 했는데, 그때 버튼과 슬라이드 메소드를 연결시키기 위해 사용했다.
