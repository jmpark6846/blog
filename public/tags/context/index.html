react context api context api 를 활용하면 여러 컴포넌트들 끼리 명시적으로 props를 통해 데이터를 넘겨주지 않고도 값을 공유할 수 있다. context 생성하면 Provider 와 Consumer 크게 두 가지 역할로 나뉜다. Provider는 값을 context를 구독하는 컴포넌트들에게 context의 변화를 알리는 역할 Consumer는 context의 변화를 구독하는 컴포넌트. 함수형 컴포넌트에서 컨텍스트에 접근하는 데 사용한다. 클래스 컴포넌트에서는 consumer를 대신하는 녀석이 <클래스 컴포넌트명>.contextType이다. 이렇게 지정하면 this.context 로 컨텍스트에 접근할 수 있다.
