호이스팅

  • 변수나 함수의 선언과 할당을 분리하여 선언을 해당 영역의 상단에 위치하도록 변경하는 걸 의미한다.
  • 정확한 의도? 는 잘모르겠으나 자바스크립트 인터프리터가 선언을 먼저 처리하고 할당은 런타임에서 처리하기 때문이라는 글을 본 것 같다.
  • var 키워드를 사용하면 호이스팅이 발생한다. let/const를 사용하면 발생하지 않음. 마찬가지로 함수선언식을 사용하면 발생하지만 함수표현식을 사용하면 발생하지 않음.
  • 최대한 발생하지 않도록 하는 것이 좋음.

var 가 아닌 let / const 를 사용해야 하는 이유

  • var 를 사용하면 스코프가 꼬일 확률이 높아진다.
  • 원래 자바스크립트에선 함수 레벨 스코프만 지원했고 블록 레벨 스코프는 지원하지 않았다.
  • 함수 내에서 블록내에서 var로 변수 선언을 하더라도 그 블록이 끝나도 함수 내에서 계속 사용할 수 있었다.
  • 하지만 let, const는 블록 레벨 스코프를 가지기 때문에 언급한 혼란이 발생하지 않는다.

렉시컬 스코프

  • 함수를 선언하는 곳에서(실행하는 곳이 아닌) 스코프가 결정되는 것을 뜻한다. 현대 대부분의 프로그래밍 언어들이 그렇듯.
