View Controller간 데이터를 주고 받을때는 각 컨트롤러의 메소드만 사용하면, 세그웨이로 prepare 메소드 실행 시 매번 새 컨트롤러 오브젝트가 생성되는 문제가 생긴다. 이때는 컨트롤러의 메소드만을 사용하는 것이 아니라 protocol을 정의 해서 Delegation Pattern을 사용한다.

2019.07.13

오늘은 Udemy로 수강중인 iOS 개발 부트캠프 강좌를 보며 계속 공부했다. Udemy 강좌는 여러개 작은 프로젝트를 만들면서 공부하도록 되어있다. 따라히기도 쉽고, 직접 개발하면서 배우는 거라 공부가 잘된다. nil 파이썬이나 자바스크립트에서 자주 사용했던 null값을, swift에선 (nil값을) 사용하지 않도록 디자인되어있다. 만약 앱 내에서 nil 값에 접근한다면 런타임에러가 난다. 이유는 프로그램을 보다 안전하게 만들기 위해서다. 이 개념이 매우 흥미로웠다. Optional ? 간단하게 말하면, 어떤 변수가 “값 or nil” 이라는 것을 나타내는 키워드이다.

Swift 로 iOS 개발을 하다가 써드파티 라이브러리로 Objective-C 코드를 사용하려면, 관련 코드를 프로젝트 폴더 내에 첨부한다. 그리고 Objective C로 Bridging Header를 추가하면 Swift 코드에서 사용할 수 있다. (xcode 자동 추가) React Native 와 Flutter를 보다가 Swift로 iOS앱 개발을 하니 확실히 다르다. 전자 쪽에선 가볍게 휙휙 만들고 얼른 결과를 보자 라는 방식이었다면 후자는 같은 걸 만들어도 좀 더 공들여 만들어야하는 느낌이다.
