2019.08.04

호이스팅 변수나 함수의 선언과 할당을 분리하여 선언을 해당 영역의 상단에 위치하도록 변경하는 걸 의미한다. 정확한 의도? 는 잘모르겠으나 자바스크립트 인터프리터가 선언을 먼저 처리하고 할당은 런타임에서 처리하기 때문이라는 글을 본 것 같다. var 키워드를 사용하면 호이스팅이 발생한다. let/const를 사용하면 발생하지 않음. 마찬가지로 함수선언식을 사용하면 발생하지만 함수표현식을 사용하면 발생하지 않음. 최대한 발생하지 않도록 하는 것이 좋음. var 가 아닌 let / const 를 사용해야 하는 이유 var 를 사용하면 스코프가 꼬일 확률이 높아진다.

2019.08.02

1. <head> 안에 <link>를 넣는 이유 페이지가 처음 로드되면 html 과 css 가 동시에 로드되어, 페이지의 컨텐츠를 우선 화면에 보여줄 수 있게 된다. 이렇게 될 경우 사용자경험도 상승하고, progressive rendering을 할 수 있게 해준다. 2. </body> 직전에 <script>를 넣는 이유 스크립트가 다운로드 되고 실행되는 동안 html 파싱을 차단. 스크립트를 아래에 두면 html을 먼저 파싱해서 사용자에게 보여줄 수 있다. 3. progressive rendering 페이지의 컨텐츠를 빠르게 표시하기 위한 기술을 말함. html과 css를 먼저 로드하여 페이지에 컨텐츠를 빨리 그리도록 한다.

2019.08.01

useEffect hook 이름처럼 side effect를 실행하는 코드를 작성하는 hook이다. useEffect의 특징 가운데 한가지는 컴포넌트의 생애주기 별로 나눠 생각하지 않고, 렌더링 된 후 실행된다는 점이다. 컴포넌트가 언마운트될 때 실행되는 함수를 작성하려면 useEffect의 리턴값으로 함수를 넘긴다. 다른 훅과 마찬가지로 useEffect는 함수 컴포넌트 안에서만 사용되어야한다. 클래스 컴포넌트에서는 사용될 수 없다. 그리고 일반 함수에서는 사용될 수 없다.
