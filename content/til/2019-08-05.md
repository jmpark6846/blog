### BST(Binary Search Tree)
> 효율적인 탐색을 위해선 찾는 방법도 중요하지만 애초에 어떻게 데이터를 저장해두는 가 도 중요하다.

- 이진 검색 트리는 특정 데이터 저장 규칙을 가진 이진 트리이다.  
  - 루트노드의 키가 왼쪽 서브 트리의 어떤 노드 키값 보다 크다.  
  - 또한 오른쪽 서브 트리의 어떤 노드 키값 보다 작다.  
  - 왼쪽과 오른쪽 서브 노트 트리도 모두 이진 검색 트리다.  
- 평균 탐색 시간은 O(log(n))이다.  최악의 경우 편향트리를 가지고 이때는 O(n)이다.  


### Heap
- 완전 이진트리 형태를 한 자료구조 중 하나이다. 보통 배열로 구현한다. 힙에는 최대힙, 최소힙 두가지가 있다.   
- 최대힙은 루트노드의 키가 모든 하위 노드들 보다 큰 값이다. 모든 서브트리에도 적용된다. 최소힙은 루트 노드가 가장 작은 값이다.  

- 최대값, 최소값을 찾는 연산은 루트 노드 값이므로 O(1) 이다. 삽입, 삭제 연산에는 O(log(n)) 이 소요된다.  
- 삽입은 트리 마지막에 원소를 넣고 루트노드 와 값을 비교하여 최상위 루트노드까지 올라가는 것이다.  
- 삭제는 최상위 루트 노드 값을 삭제한다. 마지막 노드의 값을 루트 노드로 올린뒤 자식 노드들과 비교하여 규칙이 맞을때 까지 비교해나간다.  

- 힙의 경우 연산을 하다보면 연산 시간은 O(h)가 소요된다. 편향 트리의 경우 최악의 경우인  O(n) 이 걸린다. 높이를 log(n)으로 유지 시킨다면 검색, 삽입, 삭제 연산의 시간을 O(log(n))로 유지할 수 있을 것이다. 그렇게 높이를 유지시켜줄 수 있는 트리가 바로  red-black tree이다.  

### red-black tree
- BST 중 높이를 log(n) 으로 유지시키는 규칙을 가진 트리. 따라서 삽입, 삭제, 검색에서 속도가 최대 O(log(n))으로 보장된다.  
- red-black tree 의 규칙에는 4가지가 있다.  
  1. 모든 노드는 검은색 혹은 빨간색을 가진다.  
  2. 루트 노드는 항상 검은색이다.  
  3. 인접한 두 노드의 색깔은 빨간색일 수 없다. (빨간 노드는 빨간 부모를 가질 수 없고, 빨간 자식을 가질 수 없음)  
  4. 루트 노드에서 잎노드까지의 어떤 경로에도 검은 노드의 수가 같다.  

- 이런 규칙을 준수하면 트리의 높이가 대략적으로 일정한 트리가 된다. 따라서 연산의 속도가 어느정도 보장된다.  
- 삽입시에는 우선 색깔을 변형하고, 만약 그걸로 안되면 재배열한다. 삽입한 노드의 삼촌 노드가 빨간색이면 색깔을 변형하고, 검은색이면 재배열한다.
