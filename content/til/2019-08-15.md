### bubble sort
- n개의 원소를 가진 배열을 정렬할때 한 원소와 모든 다른 원소를 비교하여 변경한다. 끝 원소까지 진행하는 정렬 방식.  
- 시간 복잡도: O(n-1 + n-2 + n-3 + ... + 1) 이므로 O(n^2)이다.   
- 공간 복잡도: 한 배열안에서만 진행하므로 O(1)이다.  

### selection sort
- 한 원소를 선택하고, 나머지 원소들과 비교하여 그 중 한 원소하고만 바꾸는 방식.  
- 시간 복잡도: 한 원소를 제외한 나머지 원소들과 모두 비교하기 때문에 역시 O(n^2)이다.  
- 공간 복잡도: 한 배열안에서만 진행하므로 O(1)이다.  

### insertion sort
- n 개의 배열 원소 중 0 부터 i-1 번째 원소는 모두 정렬되어있다고 가정한다. 그리고 i 번째 원소를 0부터 i-1 원소까지 비교하여 크거나 작을 경우 바꿔준다.  
- 시간 복잡도: O(1 + 2 + ... + n-1) 이므로 O(n^2)이다.  
- 공간 복잡도: 한 배열안에서만 진행하므로 O(1)이다.  

### merge sort
- 분할 정복을 사용하는 정렬 방식 중 하나이다.  
- n개의 원소를 가진 배열을 작은 단위로 계속 쪼갠다(분할). 더 이상 쪼갤 수 없을때 쪼갠 원소들을 차례로 합친다. 합칠때 마다 비교하여 정렬(정복)하는 방식.  
- 중간중간 쪼갠 원소들을 비교한 중간 결과를 다른 배열에 따로 저장한다.  
- 시간 복잡도: 계산해보면 쪼갰다가 합치는 각 스테이지마다 n번 비교한다. 비교하는 대상들이 이미 정렬되어 있는 상태이므로 n번만에 비교 할 수 있다. 총 logn 번의 스테이지 마다 n번 비교 하므로 O(nlogn)이다.  
- 공간 복잡도: 정렬 결과를 다른 배열에 저장하므로 O(n)이다.  

### heap sort
- binary heap을 사용하는 정렬 방식으로, 정렬하고자 하는 값을 binary heap에 넣었다가 빼면서 정렬한다.  
- binary heap은 트리의 루트노드가 최대값 혹은 최소값을 가지는 완전 이진 트리이다. 자세한 사항은 따로 정리한 글이 있으니 읽어보자.  
- binary heap에 삽입, 삭제하는 시간복잡도는 O(logn)이다.  
- 시간 복잡도: n개의 원소를 삽입했다가 삭제하는 시간 복잡도는 O(nlogn)이다.  
- 공간 복잡도: binary heap에 사용될 배열 이외에 추가적인 공간은 필요하지 않으므로 O(1) 이다.  

**til 속의 til: 시간 복잡도를 계산할때는 최악의 경우 생각해서 계산한다. 또한 어떤 숫자를 예로 들기보다는 n, n-1 같은 수치로 계산한다.**
