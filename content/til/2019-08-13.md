### 자바스크립트와 이벤트 루프
- 자바스크립트를 사용하다보면 결국 여러개의 작업을 동시에 처리하는 모습을 볼 수 있다.  
- 싱글 스레드라면서 어떻게 그럴 수 있는 걸까? 즉 **자바스크립트는 동시성(concurrency)를 어떻게 지원하는 걸까?**  
- 우선 자바스크립트가 어떻게 실행되는지 생각해보자.  
- 나는 자바스크립트로 코드를 짜서 실행을 하면 자바스크립트 엔진이 이를 지지고 볶고 다 하는 줄 알았다.  
- 그러나 자바스크립트 엔진은 콜 스택과 힙을 가지고 있을 뿐이다. 즉 코드가 실행되면 스택에 쌓아 처리하기만한다. 그렇다면 어떻게 비동기처리를 하는 걸까?  
- 답은 **자바스크립트 엔진이 실행되는 환경인 Node.js나 브라우저 등**이 맡아 처리한다는 것이다.  
- 우리가 웹 개발에서 비동기 요청을 위해 사용하는 `setTimeout`이나 `XMLHttpRequest`함수들은 자바스크립트 엔진이 아닌 브라우저에서 제공하는 Web API 영역에 정의되어 있다. 이벤트 루프나 테스크 큐도 자바스크립트 엔진 외부에 구현되어있다. Node.js 역시 마찬가지다.  
- 자바스크립트가 싱글 스레드 기반의 언어라고 하는 것은 자바스크립트 엔진에서 싱글 콜 스택을 가지기 때문이다. Node.js나 브라우저에서는 여러개의 스레드가 사용된다. **여러 스레드를 가진 Node.js나 브라우저가 싱글 스레드를 가진 자바스크립트 엔진과 상호작용하기 위해 사용하는 장치가 이벤트 루프**이다.  

### 콜스택과 테스크 큐
- 자바스크립트 엔진이 코드를 실행하면 명령문을 콜스택에 쌓는다.  
- 엔진은 스택에서 하나씩 꺼내면서 해당 명령문을 실행한다.  
- 브라우저에서 setTimeout 같은 비동기 요청이 발생하면, 자바스크립트 엔진은 setTimeout을 콜스택에 쌓는다. 그리고 스택에서 꺼내어 setTimeout을 실행한다. setTimeout은 브라우저의 Web API이므로 다른 스레드에서 명령을 실행한다.   
- Web API는 setTimeout이 끝나면 해당 콜백을 테스크 큐에 등록한다.  
- 테스크 큐에 있는 작업들은 콜스택이 비워져야만 실행된다.  
- 마이크로테스크 큐나 에니메이션 큐는 다음에 살펴보자.  

### 렌더링
- 렌더링은 브라우저의 화면이 그려지는 것을 말한다. 1초에 60프레임 정도면 최상의 상태라고 한다.  
- 렌더링도 일종의 콜백처럼 동작한다. 즉 콜스텍에 뭔가가 있으면 렌더링되지 않는다.  
- 동기적으로 시간이 오래 걸리면 작업을 하면 콜스택에서 오랫동안 남아있게되고 결국 렌더링이 제대로 일어나지 않는다. 오래 걸리는 작업을 동기로 실행하면 끝날때까지 화면이 멈추고 반응이 없는 이유이다.  
- **프론트엔드 개발을 할때 비동기로 동작하는 프로그램을 만들어야 하는 목적은 유동적인 UI로 좋은 사용자 경험을 제공하기 위해서**라는 생각이 들었다.
