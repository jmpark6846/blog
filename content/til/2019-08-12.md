### 내장 자바스크립트 객체들을 확장하면 안되는 이유
- Array 같이 자바스크립트로 이미 만들어진 객체들을 확장해서 쓰면 좋을 것 같지만 그러면 안된다.  
- 우선 확장한다는 건 prototype에 기능을 추가하는 것을 말한다. 메소드를 추가하는 것처럼  
- 만약 이렇게 되면, 브라우저가 업데이트되거나 다른 개발자들이 똑같은 이름으로 다른 기능을 추가할 경우 덮어쓰게 된다. 혹은 브라우저 기능을 덮어씌워 개발하는 경우 새로운 개발자가 해당 내용을 놓칠 수 있다.  
- 허락되는 경우는 polyfill을 만드는 경우이다. 폴리필은 오래된 브라우저에서 지원하지 않는 최신 브라우저의 기능을 지원하도록 하는 코드를 말한다.  

### == 와 === 의 차이점
- == 는 타입 변환이 필요한 경우 변환한 후 동등한지 비교한다.  
- === 는 타입 변환을 하지 않기 때문에, 타입이 다르면 동등하지 않은 것으로 판단한다.   

```javascript
1 == '1' // true  
1 == [1] // true  
1 == true // true  
0 == '' // true  
0 == '0' // true  
0 == false // true  
```

- 편의상 == 를 사용하지 않는 것이 속 편하다. 한편으로는 `1 === true`, `0 === false` 둘다 false 라는 건 조심해야겠다.  


### strict mode란
- 자바스크립트에서 안전하지 않거나 자주 실수하는 부분을 예방하기 위해 규칙을 변경하거나 추가한 모드이다. ecma2015에 추가된 기능이다.  
- 예를 들면 실수로 전역변수를 만드는 것을 불가능하게 하거나 this는 전역변수에서 undefined 로 만들거나 등  


### 동기와 비동기 함수의 차이점
- 동기 함수는 다음 코드를 실행하기 위선 현재 코드가 완료되어야만 한다. 프로그램의 순서가 정확하게 지켜진다. 만약 중간에 실행 시간이 오래 걸리는 코드가 있다면 완료될때까지 프로그램의 실행이 멈춘다.  
- 비동기 함수는 호출되면 바로 다음 코드로 진행된다. 그리고 비동기 함수는 백그라운드에서 실행되고 실행 완료되면 콜백이 실행된다. 콜백은 비동기 함수를 실행할때 미리 받는다.  
- 계속해서 코드가 진행되기 때문에 프로그램이 멈추지 않는다. 주로 무거운 작업들에 사용된다. 데이터베이스 쿼리 실행, pdf 파일 추출 등
